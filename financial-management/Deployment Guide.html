ต้องการ GitHub Account

ถ้ายังไม่มีให้สมัครฟรีที่ github.com ก่อน

วิธี Deploy ไป Vercel

1

สร้าง GitHub Repository

ไปที่ github.com/new → ตั้งชื่อ class-fund-67 → Private → Create

2

อัปโหลดไฟล์โปรเจกต์

คลิก "Add file" → "Upload files" แล้วลากไฟล์ทั้งหมดจากโปรเจกต์นี้ขึ้นไป (Financial Management.html, *.jsx, data.js, vercel.json)

3

Connect Vercel

ไปที่ vercel.com/new → "Import Git Repository" → เลือก repo ที่สร้างไว้ → Deploy

4

ตั้ง Root Directory

ใน Vercel Settings → ตั้ง "Output Directory" เป็น ./ และไฟล์หลักเป็น Financial Management.html

5

เสร็จ! ได้ URL ฟรี

Vercel จะให้ URL เช่น class-fund-67.vercel.app — แชร์ให้นักศึกษาได้เลย

สิ่งที่ต้องทำใน Phase 2

1

สร้าง Supabase Project

ไปที่ supabase.com → New Project → ตั้งชื่อ class-fund-67 → Region: Singapore (ap-southeast-1)

2

รัน SQL Schema

ไปที่ SQL Editor ใน Supabase Dashboard → วาง supabase-schema.sql (ไฟล์ที่สร้างไว้) → Run

3

เพิ่ม Supabase JS Client ในเว็บ

เพิ่ม @supabase/supabase-js แล้วแทนที่ข้อมูล mock ใน data.js ด้วย Supabase queries

// ตัวอย่าง: ดึงรายชื่อนักศึกษา
import { createClient } from '@supabase/supabase-js'
const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY)
const { data } = await supabase
  .from('students')
  .select('*')
  .order('id')
4

ตั้ง Auth สำหรับ Login

ใช้ Supabase Auth แทน mock login — นักศึกษา login ด้วย student ID, admin login ด้วย email+password ผ่าน Supabase Dashboard

5

เพิ่ม Environment Variables ใน Vercel

ใน Vercel → Settings → Environment Variables → เพิ่ม SUPABASE_URL และ SUPABASE_ANON_KEY (หาได้ใน Supabase Settings)

📁 ไฟล์ที่สร้างไว้ให้แล้ว

ไฟล์ใช้ทำอะไรสถานะ
vercel.json Config สำหรับ deploy บน Vercel (MIME types สำหรับ JSX) พร้อมใช้
netlify.toml Config สำหรับ deploy บน Netlify (ทางเลือก) พร้อมใช้
supabase-schema.sql SQL สร้าง tables ทั้งหมด + ข้อมูลเริ่มต้น Phase 2

🗺 ขั้นตอนต่อไปที่แนะนำ

  1. สมัคร GitHub + Vercel (ฟรี ใช้เวลา 5 นาที)
  2. อัปโหลดไฟล์โปรเจกต์นี้ขึ้น GitHub repo
  3. Connect Vercel → ได้ URL สาธารณะ แชร์ให้นักศึกษาได้ทันที
  4. สมัคร Supabase (ฟรี)
  5. รัน supabase-schema.sql เพื่อสร้าง database
  6. แจ้งให้นักพัฒนา (หรือผม) เชื่อม Supabase กับเว็บ
